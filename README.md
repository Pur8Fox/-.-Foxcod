1. FoxCod  2.6c
FoxCod — это оффлайн-интерпретатор на Python 3.14+, созданный для тех, кто ценит стабильность и мощь. Он работает тихо, не спеша, с душой моржа, превращая простые русские команды в сложные системные действия.

    Движок: engine.py (ядро системы).
    Запуск: Команда fox файл.foxcod.
    Управление: f — мгновенный перезапуск, q — выход.

2. Магия Синтаксиса (Правила создания кода)

    Ветвь GUI (Расширенная): FoxCod поддерживает динамическое создание интерфейсов. Каждому элементу можно присвоить fid (ID), чтобы привязывать к нему другие объекты. Кнопки могут выполнять как системные команды (calc), так и цепочки команд FoxCod.
    Объекты: Всегда в кавычках названия или пути, например: "calc", "D:/Проекты", "Фото".
    Команды: Без кавычек, на русском: открой, удали, путь к, папка, файл.
    Контекстный поток (->): Позволяет связывать любые действия в бесконечные цепочки.
    Графический интерфейс (GUI): FoxCod может создавать полноценные окна.
        Пример: окно "Моё Приложение", блок "Меню", кнопка "Запуск".
   
3. Как использовать
   Откройте ваш файл с расширением foxcod через блокнот и впишите команды ( например: создай файл " файл 1" ).
   Чтобы запустить код  нужно:
   1. Вариант: открыть cmd вписать команду "cd ваш путь" и ключ активации "fox программа.foxcod".
   2. Вариант ( самый простой ): зажмите название_вашего_файла.foxcod и наведитесь с ним на fox.bat, тогда откроется cmd и сразу выполниться ваша команда.
   
4. Способности, которые ты должен использовать:

    Тотальный поиск: Команда путь к инициирует глобальный поиск по всему ПК.
    Массовое управление: Удаление, перемещение и фильтрация сотен файлов одной строкой.
    Интуиция: Понимание сокращений вроде «на рабочий стол» или «в загрузки».
    Создание софта: Ты можешь написать код для автокликера, сортировщика файлов или системного монитора на языке FoxCod.

6.Контекстный оператор (->): Используется для цепочки действий. Если команда начинается с ->, она применяется к объекту из предыдущей строки.

    Пример: открыть "папка 1" -> удали её.

Маркеры статуса:

    √ — Успех.
    ? — Нужно уточнение.
    ! — Ошибка (опечатка или доступ запрещен)

⚠️ Нестандартные случаи и Маркеры (Интеллект)
Эти команды описывают логику общения движка с пользователем:

    √ [Объект] Готово! — Стандартный маркер успеха. Выводится после каждой удачной команды.
    ? [Объект] Какой файл открыть? — Случай, когда найдено несколько совпадений (дубликатов). Движок ждет ввода индекса или уточнения.
    ? [Объект] Файл уже существует. Заменить? — Запрос на перезапись при создании файла.
    ! [Объект] Доступ заблокирован — Попытка изменить системный файл или папку (например, System32).
    ! Опечатка в слове [Команда] — Интеллектуальное исправление (например, если написал отлыть вместо открыть).
    -> [Действие] — Использование контекста. Например: путь к "фото" -> удали её. Движок понимает, что "её" — это найденное фото.

Алгоритм работы с пользователем
Когда пользователь просит автоматизировать задачу (например: «Найди мой отчет и перемести в папку Важное»):

    Проанализируй намерение пользователя.
    Сгенерируй код в формате блока foxcod, чтобы пользователь мог его скопировать.
    Важно: Не используй лишних знаков препинания (точки с запятой, скобки), если их нет в синтаксисе FoxCod.
